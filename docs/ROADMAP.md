# 로드맵

각 Sprint는 1주 안팎의 작은 결과물을 목표로 하며, 완료 기준을 충족한 뒤 다음 단계로 간다. 일정과 provider 선택은 검증 결과에 따라 조정한다.

## Sprint 0 — 기반 정리와 계약

- 기본 샘플 화면을 대체할 정보 구조와 핵심 라우트 설계
- 도메인 타입, repository/provider interface, 오류 코드 정의
- 합성 OCR fixture와 대표 이미지 테스트 세트 준비
- 디자인 토큰과 공용 버튼/상태 컴포넌트의 최소 기준 수립

완료 기준: Expo Go에서 홈→새 단어장→mock 교정→카드 미리보기 흐름을 탐색하고 lint/typecheck가 통과한다.

## Sprint 1 — 사진 입력

- 카메라 촬영 및 앨범 1장 선택
- 권한 요청/거부/취소/재시도 UX
- 앱 전용 파일 저장, 이미지 크기 제한과 메타데이터 최소화
- 사진 개인정보 안내와 원본 보존 선택

완료 기준: iPhone 실기기에서 촬영과 선택 결과가 작업 초안으로 복원되고, Android 호환 차이가 기록된다.

## Sprint 2 — OCR 교정 경험

- mock `OcrProvider`와 작업 상태 저장
- 후보 목록, 원문 위치, 수정·삭제·추가, 중복 경고
- 결과 없음, 부분 실패, 작업 재개
- 대표 데이터셋으로 교정 UX 측정

완료 기준: 외부 OCR 없이도 사진 작업부터 사용자 확정까지 완주하고 앱 재실행 후 draft가 남는다.

## Sprint 3 — 실제 OCR과 빌드 전환

- 온디바이스/서버 OCR 기술 spike 및 개인정보·비용·정확도 비교
- provider 결정 기록, adapter 구현, runtime validation
- 네이티브 모듈 채택 시 development build/config plugin 구성
- timeout, rate limit, 취소, 재시도 처리

완료 기준: 4개 학습 범주 대표 세트에서 목표 품질을 측정하고 사용자가 모든 오인식을 교정할 수 있다.

## Sprint 4 — 로컬 단어장

- SQLite migration과 단어장/단어/OCR 모델 구현
- 사용자 확정 transaction, 중복 처리, 검색/편집/삭제
- 앱 재실행과 오프라인 데이터 복원
- 원본/파생 파일 정리 정책 구현

완료 기준: 네트워크 없이 단어장을 생성·편집·삭제하고 강제 종료 후 데이터가 일관되다.

## Sprint 5 — 단어 정보와 이미지

- 사전 provider로 뜻·품사·발음·예문 조회 및 직접 편집
- 무료 이미지 검색, 출처/작가/라이선스 표시와 저장
- 검색 결과 부적합 상태 이후에만 AI 이미지 생성 fallback
- 필드별 부분 실패와 재시도, 의학 학습 주의 표시

완료 기준: 외부 서비스 하나가 실패해도 단어 저장이 가능하고, 선택한 검색 이미지의 출처 보유율이 100%다.

## Sprint 6 — 플래시카드와 간격 반복

- 오늘의 복습 큐, 카드 앞/뒤, 네 단계 평가
- 버전이 있는 간격 반복 알고리즘
- 학습 기록과 일정의 원자적 저장, 평가 실행 취소
- 접근성, 모션 감소, 대형 글자 대응

완료 기준: 단위 테스트로 일정 계산을 재현하고 오프라인 복습 후 재실행해도 due 상태가 유지된다.

## Sprint 7 — MVP 안정화와 베타

- 핵심 흐름 회귀 테스트와 실기기 성능 측정
- 삭제, 로그 redaction, API 제한, 이미지 출처, 개인정보 문구 점검
- Android 호환 수정, crash/empty/loading 상태 정리
- 베타 성공 지표 수집(민감 payload 제외)과 피드백 채널

완료 기준: 제품 성공 기준을 측정할 수 있고 차단급 결함 없이 제한 베타를 배포할 준비가 된다.

## MVP 이후

- Supabase Auth, RLS, Storage, outbox 기반 다중 기기 동기화
- 충돌 해결 UI와 로컬 게스트 데이터 계정 병합
- 알림, 단어장 가져오기/내보내기, 품질 개선
- 공유/협업, 결제, 다중 페이지는 별도 제품 검증 후 결정
