# LinkVoca UI/UX 개요

## 문서 범위

이 문서는 iPhone 우선·Android 호환 MVP의 화면 구조와 공통 경험을 정의한다. 현재 앱은 기본 Expo 템플릿 단계이며 아래 내용은 구현 완료 상태가 아닌 설계 기준이다. PDF 가져오기, AI 이미지 생성, 계정 동기화는 향후 확장을 고려해 진입점만 설계하고 MVP 완료 조건에는 넣지 않는다.

## UX 목표

1. 사진 선택부터 편집 가능한 OCR 후보까지 사용자가 길을 잃지 않게 한다.
2. OCR을 자동 정답이 아닌 초안으로 명확히 보여 주고 저장 전 검토를 필수로 한다.
3. 영어 초보자도 “무엇을 고치고 다음에 무엇을 눌러야 하는지” 이해하게 한다.
4. 외부 조회가 실패해도 사진·교정 내용·학습 기록을 로컬에 보존한다.
5. 한 화면의 주요 행동을 하나로 제한하고 작은 iPhone에서도 핵심 버튼을 항상 찾게 한다.
6. 이미지 출처와 의료·간호 정보 주의를 학습 흐름을 방해하지 않으면서 확인 가능하게 한다.
7. 오늘 할 복습량과 완료감을 짧고 명확하게 제공한다.

## 핵심 사용자 시나리오

- 수능 교재 한 페이지를 촬영하고 오인식 두 개를 고쳐 새 단어장에 저장한다.
- 토익 스크린샷 여러 장을 차례로 추가하고 중복 단어를 제외한다.
- 간호학생이 의학 용어를 직접 추가하고 일반 뜻과 의학 뜻을 구분해 확인한다.
- 검색 이미지 후보 중 출처가 분명한 이미지를 선택해 연상 카드로 만든다.
- 통학 중 오프라인으로 오늘의 복습을 끝내고 앱을 다시 열어 이어서 학습한다.

## 정보 구조

```text
앱
├─ 홈
│  ├─ 오늘의 복습
│  ├─ 최근 단어장
│  └─ 단어 가져오기
│     ├─ 카메라
│     ├─ 앨범
│     └─ PDF (출시 후, 비활성 안내)
├─ 단어장
│  ├─ 단어장 목록
│  ├─ 단어장 상세
│  │  ├─ 단어 상세
│  │  ├─ 단어 수동 추가
│  │  └─ 플래시카드/퀴즈
│  └─ 단어장 생성/편집
├─ 학습
│  ├─ 오늘의 복습
│  ├─ 이어서 학습
│  └─ 학습 통계
└─ 설정
   ├─ 사진 및 데이터
   ├─ 접근성/화면
   ├─ 학습 설정
   └─ 정보/의료 정보 안내
```

## 하단 탭 구조

MVP는 네 탭을 사용한다.

| 탭 | 기본 화면 | 역할 | 배지 |
|---|---|---|---|
| 홈 | 홈 | 다음 행동과 최근 작업의 출발점 | 없음 |
| 단어장 | 단어장 목록 | 단어장 탐색·관리 | 없음 |
| 학습 | 오늘의 복습 | 예정 카드와 학습 진입 | 오늘 남은 카드 수, 99+ 제한 |
| 설정 | 설정 | 데이터·권한·표시·안내 | 필요한 조치가 있을 때만 점 표시 |

촬영은 탭으로 만들지 않는다. 홈과 단어장 상세의 강조 버튼에서 동일한 “단어 가져오기” bottom sheet를 열어 카메라/앨범을 선택한다. 처리·교정·학습 중에는 하단 탭을 숨겨 집중과 실수 방지를 돕는다.

## 주요 화면 간 이동 흐름

```text
홈 → 단어 가져오기 → 촬영/앨범 → 미리보기·자르기 → OCR 처리
   → OCR 결과 검토 → 기존 단어장 선택 또는 새 단어장 생성 → 단어장 상세

홈/학습 → 오늘의 복습 → 플래시카드 → 완료 요약 → 학습/통계

단어장 목록 → 단어장 상세 → 단어 상세 → 연상 이미지 선택
                          └→ 플래시카드 또는 퀴즈
```

- 시스템 뒤로 가기는 입력 내용을 버리지 않는다. 사진 작업과 OCR 교정에서 이탈하면 `임시 저장 후 나가기`를 기본 행동으로 제공한다.
- 완료 후에는 깊은 화면 스택을 되짚지 않고 단어장 상세나 학습 탭의 기준 화면으로 교체 이동한다.

## 첫 사용자의 온보딩 흐름

온보딩은 권한을 미리 요청하지 않는 3단계다.

1. **가져오기:** “사진 속 영어 단어를 빠르게 옮겨요.” 카메라·앨범·향후 PDF를 설명한다.
2. **직접 확인:** “저장 전, 틀린 단어를 직접 고칠 수 있어요.” 자동 저장하지 않음을 강조한다.
3. **기억하기:** “이미지 카드와 복습 일정으로 오래 기억해요.” 로컬 우선 및 의료 정보 참고용 안내 링크를 제공한다.

`시작하기` 후 홈으로 이동한다. 카메라 권한은 사용자가 `사진 촬영`을, 앨범 권한은 `앨범에서 선택`을 처음 누른 시점에 목적 문구를 보여 준 뒤 요청한다. 온보딩은 설정에서 다시 볼 수 있고 건너뛰기를 제공한다.

## 상태 설계 원칙

### 오류

- 오류가 난 대상과 보존된 내용을 먼저 말한다: “이미지는 기기에 저장했어요. 단어 추출만 다시 시도해 주세요.”
- 내부 코드 대신 사용자가 할 수 있는 행동을 제공한다: `다시 시도`, `직접 추가`, `나중에 계속`.
- 네트워크 오류는 로컬 작업 실패처럼 표현하지 않는다. 외부 정보 필드만 `불러오지 못함`으로 둔다.
- 전체 화면 오류는 다음 단계로 갈 방법이 없을 때만 사용한다. 필드 단위 실패는 해당 카드 안에서 처리한다.

### 빈 상태

- 무엇이 비었는지, 왜 유용한지, 첫 행동 하나를 함께 제시한다.
- 빈 단어장에는 `단어 가져오기`와 보조 행동 `직접 추가`를 제공한다.
- 검색 결과 없음은 실패와 구분하며, 검색어 수정 후 향후 AI 생성 선택지를 안내한다.

### 로딩

- 300ms 미만 작업에는 불필요한 spinner를 피한다.
- 예상 시간이 긴 OCR에는 단계 문구(`이미지 준비 중 → 단어 찾는 중 → 결과 정리 중`), 진행 표시, 안전한 취소를 제공한다.
- 이미 보이는 콘텐츠는 skeleton으로 전부 지우지 않고 해당 필드만 갱신한다.
- 중복 제출을 막되 취소·뒤로 가기 경로는 유지한다.

## 접근성 원칙

- 모든 터치 대상은 최소 44×44pt이며 인접 대상 사이에 8pt 이상 여유를 둔다.
- Dynamic Type에서 텍스트 확대와 줄바꿈을 허용하고 고정 높이 텍스트 컨테이너를 피한다.
- 색상만으로 선택·오류·OCR 낮은 신뢰도를 전달하지 않는다. 문구, 아이콘, 접근성 상태를 함께 쓴다.
- VoiceOver/TalkBack 탐색 순서는 화면의 시각적 읽기 순서와 같게 한다.
- 아이콘 버튼에는 “후보 단어 삭제”, “카드 앞면 보기”처럼 대상과 행동을 포함한 label을 준다.
- OCR 완료, 카드 뒤집힘, 저장 성공은 접근성 알림으로 전달한다.
- 모션 감소 설정에서는 카드 뒤집기와 화면 전환을 페이드 또는 즉시 전환으로 대체한다.
- 라이트·다크 모드 모두 텍스트와 상태 색 대비를 검증하고 이미지 위 텍스트를 직접 겹치지 않는다.
- 작은 iPhone에서는 하단 고정 CTA가 콘텐츠를 가리지 않도록 safe area와 키보드 높이를 반영한다.
