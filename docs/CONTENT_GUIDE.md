# 콘텐츠 가이드

## 한국어 문구 작성 원칙

- 존댓말 `-해요/-하세요`를 기본으로 사용하고 한 화면 안에서 문체를 섞지 않는다.
- 초보자가 이해할 수 있는 일상어를 먼저 쓴다. `OCR`은 첫 노출에서 “사진 속 단어 찾기(OCR)”로 설명하고 이후 “단어 찾기”로 쓴다.
- 제목은 상태나 목적, 본문은 이유, 버튼은 다음 행동을 말한다.
- 한 문장은 가능하면 25자 안팎으로 쓰고 긴 설명은 두 문장으로 나눈다.
- `완료`, `확정`, `저장`을 구분한다. OCR 후보는 `검토 완료`, 단어장 반영은 `단어 저장`, 학습 세션은 `복습 완료`다.
- 외부 결과는 “추천”, “찾은 결과”, “확인 필요”로 표현하며 “정답”, “완벽한 분석”이라 하지 않는다.
- 사진은 기본적으로 기기에 저장된다는 점과 서버 전송 여부를 행동 가까이에서 설명한다.

## 버튼 문구

| 상황 | 권장 | 피할 문구 |
|---|---|---|
| 온보딩 완료 | `시작하기` | `확인` |
| 입력 방식 | `사진 촬영`, `앨범에서 선택` | `업로드`만 단독 사용 |
| OCR 시작 | `단어 찾기` | `실행`, `제출` |
| 후보 확정 | `12개 단어 검토 완료` | `다음` |
| 저장 | `이 단어장에 저장` | `OK` |
| 재시도 | `단어 다시 찾기` | `Retry` |
| 수동 대안 | `직접 단어 추가` | `건너뛰기` |
| 파괴적 행동 | `사진 삭제`, `단어장 삭제` | `삭제`만 단독 사용 |
| 학습 평가 | `다시`, `어려움`, `보통`, `쉬움` | 의미 없는 숫자 점수 |

주요 버튼에는 대상 또는 결과를 포함한다. 개수가 바뀌는 행동에는 `8개 단어 저장`처럼 현재 값을 반영한다.

## 권한 요청 문구

### 카메라 사전 안내

- 제목: `교재를 촬영할까요?`
- 본문: `사진에서 영어 단어를 찾기 위해 카메라 접근이 필요해요. 촬영한 사진은 기본적으로 이 기기에 저장돼요.`
- 버튼: `카메라 사용하기` / `나중에`

### 앨범 사전 안내

- 제목: `앨범에서 사진을 고르세요`
- 본문: `선택한 사진만 단어 찾기에 사용해요. 다른 사진에는 접근하지 않아요.`
- 버튼: `사진 선택하기` / `취소`

### 권한 거부 후

- 제목: `카메라 권한이 꺼져 있어요`
- 본문: `설정에서 카메라를 허용하거나, 앨범의 사진을 사용할 수 있어요.`
- 버튼: `설정 열기` / `앨범에서 선택`

권한을 주면 더 안전하거나 성능이 좋아진다고 과장하지 않는다. 시스템 권한창이 뜨기 직전에만 사전 안내를 보여 준다.

## 오류 메시지 작성법

`무슨 일이 있었는지 → 무엇이 보존됐는지 → 할 수 있는 행동` 순서로 쓴다.

| 상황 | 실제 문구 | 행동 |
|---|---|---|
| 일반 네트워크 | `인터넷에 연결되지 않았어요. 수정한 단어는 기기에 저장했어요.` | `다시 연결` / `나중에 계속` |
| 사전 정보 실패 | `뜻과 예문을 불러오지 못했어요. 단어는 먼저 저장할 수 있어요.` | `다시 불러오기` / `직접 입력` |
| 저장 공간 부족 | `기기의 저장 공간이 부족해 사진을 보관하지 못했어요.` | `공간 확인` / `사진 없이 계속` |
| 알 수 없는 오류 | `작업을 마치지 못했어요. 임시 내용은 그대로 보관했어요.` | `다시 시도` / `홈으로` |

`오류가 발생했습니다`, `잘못된 요청`, 숫자 오류 코드만 보여 주지 않는다. 진단 코드는 도움말 상세나 로그에 분리한다.

## OCR 실패 안내

### 단어를 찾지 못함

- 제목: `영어 단어를 찾지 못했어요`
- 본문: `글자가 선명하게 보이도록 다시 자르거나 촬영해 보세요. 단어를 직접 추가할 수도 있어요.`
- 버튼: `이미지 다시 확인` / `직접 단어 추가`

### 처리 중 실패

- 제목: `단어 찾기를 마치지 못했어요`
- 본문: `선택한 사진은 기기에 남아 있어요. 같은 사진으로 다시 시도할 수 있어요.`
- 버튼: `다시 시도` / `나중에 계속`

### 일부만 인식

- banner: `확인이 필요한 단어가 있어요. 주황색 표시를 먼저 살펴보세요.`

## 이미지 출처 표시

- 기본 형식: `사진: {작가명} · {제공자}`
- 라이선스: `{라이선스명} · 원본 보기`
- 작가 미상: `작가 정보 없음 · {제공자}`로 표시하되 라이선스가 불명확하면 선택을 막는다.
- 생성 이미지: `AI로 생성한 이미지`와 생성 provider를 표시한다.
- 출처 영역은 최소 14pt 본문을 사용하고 말줄임 뒤에도 `출처 자세히 보기`로 전체 정보를 확인할 수 있어야 한다.

예시:

> 사진: Jane Kim · Example Images
>
> CC BY 4.0 · 원본 보기

## 의료·간호 영어 정확성 주의 문구

짧은 인라인 문구:

> 학습 참고용 설명이에요. 진단이나 처치 판단에는 사용하지 마세요.

상세 안내:

> 간호·의학 단어의 뜻과 예문은 학습을 돕기 위한 정보예요. 약어와 임상 표현은 상황에 따라 의미가 달라질 수 있어요. 실제 환자 진료나 과제에는 교재, 교수자, 공인 의학 자료를 함께 확인하세요.

AI가 관여한 필드는 `AI 제안 · 확인 필요`를 표시한다. “의학적으로 검증됨”은 실제 검수 체계와 근거가 없으면 사용하지 않는다.

## 사용자를 탓하지 않는 원칙

- `사진을 잘못 찍었습니다` 대신 `글자가 흐리게 보여요`라고 쓴다.
- `올바른 단어를 입력하세요` 대신 `영어 단어만 입력할 수 있어요`라고 조건을 설명한다.
- `권한을 거부했습니다` 대신 `카메라 권한이 꺼져 있어요`라고 상태를 말한다.
- `학습을 놓쳤어요` 대신 `오늘 다시 시작할 수 있어요`라고 회복 행동을 준다.
- 경고로 사용자를 압박하거나 연속 학습 기록 손실을 위협하지 않는다.

## 주요 화면별 실제 예시

| 화면 | 제목/본문 | 주요 버튼 |
|---|---|---|
| 온보딩 1 | `사진에서 단어를 찾아요` / `교재나 스크린샷을 다시 입력하지 않아도 돼요.` | `다음` |
| 홈, 첫 사용 | `첫 단어장을 만들어 볼까요?` / `사진 한 장으로 시작할 수 있어요.` | `단어 가져오기` |
| 홈, 복귀 | `이어서 정리할 사진이 있어요` / `7개 후보를 아직 검토하지 않았어요.` | `검토 이어하기` |
| 촬영 | `단어가 선명하게 보이게 맞춰 주세요` / `이름이나 개인정보는 사진에서 제외해 주세요.` | `촬영` |
| 미리보기 | `단어가 있는 부분만 남겨 주세요` | `이 이미지 사용` |
| OCR 처리 | `사진에서 영어 단어를 찾고 있어요` / `앱을 나가도 임시 작업은 기기에 남아요.` | `취소` |
| OCR 검토 | `찾은 단어를 확인하세요` / `저장 전에 틀린 글자를 고칠 수 있어요.` | `12개 단어 검토 완료` |
| 단어장 생성 | `어디에 저장할까요?` / `새 단어장 이름과 학습 분야를 정하세요.` | `단어장 만들기` |
| 단어장 상세 | `토익 800 핵심` / `42개 단어 · 오늘 8개 복습` | `학습 시작` |
| 단어 상세 | `administer` / `동사 · 투여하다, 관리하다` | `수정` |
| 이미지 선택 | `administer와 연결할 이미지를 고르세요` / `무료 검색 결과를 먼저 보여 드려요.` | `이 이미지 사용` |
| 복습 | `오늘 8개가 기다리고 있어요` | `복습 시작` |
| 완료 | `오늘 복습을 마쳤어요` / `8개 중 6개를 기억했어요.` | `학습으로 돌아가기` |
| 빈 통계 | `아직 학습 기록이 없어요` / `카드 1개부터 시작하면 여기에 변화가 보여요.` | `첫 학습 시작` |
| 설정 | `사진 보관` / `단어 저장 후 원본 사진을 자동으로 삭제해요.` | toggle label `자동 삭제` |
