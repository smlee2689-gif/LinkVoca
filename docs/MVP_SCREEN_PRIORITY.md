# MVP 화면 우선순위

## 분류 기준

- **반드시 MVP에 포함:** 사진 1장에서 검토 가능한 단어를 만들고 로컬 단어장에 저장·복습하는 핵심 고리를 완성한다.
- **MVP 후반:** 핵심 고리 안정화 후 유지율·관리·품질을 높인다. 베타 출시에 포함할 수 있으나 앞 단계 차단 요소는 아니다.
- **출시 후:** 계정/확장 입력/고급 학습처럼 기술 또는 제품 검증이 더 필요한 기능이다.

## 반드시 MVP에 포함

| 화면/상태 | 이유 | 최소 완료 범위 |
|---|---|---|
| 홈 | 모든 핵심 행동의 짧은 출발점 | 최근 작업, 복습 요약, 가져오기 CTA |
| 사진 촬영·앨범 선택 | 핵심 입력 | 각 1장, 권한·취소 처리 |
| 미리보기·자르기 | OCR 품질과 개인정보 최소화 | 회전, 다시 선택, 사용 영역 확인 |
| OCR 처리 중 | 긴 작업의 신뢰 | 단계 문구, 취소, 로컬 보존 안내 |
| OCR 결과 검토 | 제품의 안전 핵심 | 수정·삭제·추가, 선택, 확정 전 필수 검토 |
| 단어장 생성·목록·상세 | 로컬 결과 관리 | 분야, 이름, 단어 행, 수동 추가 |
| 단어 상세 | 정보 확인·교정 | 뜻/품사/발음/예문/연상 설명, 부분 실패 |
| 플래시카드 | 핵심 학습 | 앞/뒤, 진행률, 종료 후 보존 |
| 오늘의 복습 | 간격 반복 진입 | 예정 카드, 네 단계 평가 연결 |
| 권한 거부·OCR 실패·네트워크 오류·빈 상태 | 흐름 회복 | 다음 행동과 보존 데이터 설명 |

## MVP 후반

| 화면/기능 | 이유 | 최소 완료 범위 |
|---|---|---|
| 연상 이미지 선택 | 기억 가치 강화, provider 결정 필요 | 무료 검색 우선, 출처·라이선스 공간 |
| 퀴즈 | 학습 방식 확장 | 객관식 한 유형과 결과 |
| 학습 통계 | 동기 부여 | 오늘/7일 학습량, 정확도, 연속 일수 |
| 설정 | 운영·접근성 완성 | 원본 보존, 테마, 온보딩 다시 보기, 안내 |
| 여러 장 연속 업로드 | 대량 입력 효율 | 순차 큐, 장별 실패/재시도, 중복 처리 |

## 출시 후

| 화면/기능 | 미루는 이유 |
|---|---|
| PDF 가져오기·페이지 선택 | MVP 제외 범위이며 파일·다중 페이지 처리 검증 필요 |
| AI 이미지 생성 UI | provider, 비용, 안전, 약관과 일일 제한 미정 |
| 계정·Supabase 동기화·충돌 해결 | 로컬 MVP 이후 결정 |
| 공유 단어장·협업 | 개인정보와 권한 모델 필요 |
| 고급 퀴즈·말하기·작문 | 핵심 OCR→복습 가치 검증 후 판단 |
| iPad·웹 전용 레이아웃 | 공식 지원 범위 미정 |

## 권장 구현 순서

각 항목은 독립 검토 가능한 작은 작업 단위다.

1. 홈의 정적 정보 구조와 `단어 가져오기` bottom sheet
2. mock 데이터 기반 OCR 결과 검토 목록 한 행: 선택, 편집, 삭제
3. OCR 검토의 후보 추가·전체 선택·중복 경고·하단 확정 CTA
4. 기존 단어장 선택/새 단어장 생성 sheet와 완료 이동
5. 단어장 목록의 빈 상태와 단어장 상세 기본 목록
6. 단어 수동 추가와 단어 상세의 편집 가능한 최소 필드
7. 사진 촬영/앨범 권한 전 안내와 거부 화면
8. 이미지 미리보기·자르기 및 작업 임시 저장 안내
9. mock OCR 처리 단계·취소·실패·재시도 연결
10. 앱 재실행 후 미완료 OCR 작업 banner와 이어하기
11. 플래시카드 앞면·뒷면·진행률·중도 종료
12. 오늘의 복습 목록과 네 단계 평가 UI
13. 오프라인/네트워크 부분 실패 banner와 필드별 재시도
14. 이미지 검색 결과 grid, 상세 출처, 선택 저장
15. 여러 장 업로드 큐와 장별 상태
16. 퀴즈 한 유형, 통계 최소 지표, 설정
17. Dynamic Type, VoiceOver/TalkBack, 모션 감소, 다크 모드 회귀 검증
18. Android 권한·뒤로 가기·edge-to-edge·키보드 대응

## MVP 첫 구현 화면 추천

첫 구현은 **mock 데이터 기반 OCR 결과 검토 화면**을 추천한다. LinkVoca의 차별점인 “자동 저장 전 사용자 검토”를 가장 빨리 검증하며, `WordCandidateRow`, 상태 배지, 편집 입력, 하단 CTA 같은 공용 UI 계약도 함께 정할 수 있다. 카메라나 OCR provider가 없어도 Expo Go에서 테스트할 수 있고 이후 사진 입력·단어장 저장 양쪽의 데이터 계약을 구체화한다.
