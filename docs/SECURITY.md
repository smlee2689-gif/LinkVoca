# 보안·개인정보·콘텐츠 안전

## 보호 대상과 위협

보호 대상은 교재 사진, 사진에서 추출한 텍스트, 사용자의 단어장/학습 기록, 계정 토큰, 외부 API 자격 증명이다. 주요 위험은 사진의 의도치 않은 외부 전송, 앱 번들 속 비밀 키 노출, 로그/백업 잔존, 다른 사용자 데이터 접근, 출처 없는 이미지 재사용, 부정확한 의학 설명의 오용이다.

## 사진과 개인정보

- 촬영/앨범 권한은 기능 진입 시 맥락과 함께 요청하며, 거부해도 앱 설명과 설정 이동 경로를 제공한다.
- 사진에는 이름, 학교, 환자 정보, 필기, 얼굴이 포함될 수 있다고 촬영 전에 안내한다. 의료 현장의 환자 기록 촬영을 금지하는 문구를 명확히 둔다.
- 기본 처리는 로컬 우선이다. 서버 OCR을 채택하면 전송 전에 제공자, 처리 목적, 보관 여부를 알리고 별도 동의를 받는다.
- 가능하면 기기에서 페이지 영역을 자르고 해상도/메타데이터를 최소화한 뒤 전송한다. EXIF 위치 정보는 전송하지 않는다.
- 원본 자동 삭제 또는 보존 옵션을 제공하고, 사용자가 단어장/작업을 삭제하면 관련 로컬 파일과 캐시도 삭제한다.
- 원본 사진, OCR 전체 텍스트, 단어 목록을 analytics, crash log, 개발 console에 남기지 않는다.

## API 키와 인증

- `EXPO_PUBLIC_*` 환경 변수는 공개 값이다. Supabase anon key처럼 RLS와 함께 쓰는 공개 클라이언트 설정만 허용한다.
- OCR, 이미지 검색, AI 생성, Supabase service role 비밀 키는 앱 코드·`.env`·EAS Update bundle에 넣지 않는다. 서버 프록시 또는 Supabase Edge Function의 secret으로 보관한다.
- 프록시는 사용자 인증, 요청 크기/형식 검증, rate limit, provider별 비용 한도, timeout을 적용한다.
- Supabase 도입 시 모든 사용자 테이블과 Storage bucket에 RLS를 켜고 소유권 정책을 자동 테스트한다. service role은 모바일 클라이언트에서 절대 사용하지 않는다.
- 토큰은 플랫폼 보안 저장소를 사용하고 로그에 헤더·URL query·응답 원문을 기록하지 않는다.
- 키 노출이 의심되면 즉시 폐기/회전하고 저장소 이력과 배포 번들을 점검한다.

## 이미지 검색, 저작권과 출처

- 무료 API라는 이유만으로 이미지 사용 권한을 가정하지 않는다. 상업 이용, 수정, 앱 내 캐시 허용 조건을 provider 약관에서 확인한다.
- 검색 이미지 선택 시 제공자, 작가, 원본 페이지, 라이선스 이름/URL, attribution 문구를 저장하고 카드 상세에서 접근 가능하게 한다.
- hotlink 금지나 다운로드 금지 조건을 존중한다. 라이선스 메타데이터가 없거나 불명확하면 저장 대상으로 제공하지 않는다.
- 삭제 요청과 provider URL 만료에 대응할 수 있도록 출처별 제거 경로를 둔다.
- AI 생성 이미지는 검색 결과가 부적합할 때만 생성하며 “AI 생성”으로 표시한다. 사용자의 사진이나 개인정보가 prompt로 넘어가지 않게 한다.

## 간호·의학 영어 정확성

- LinkVoca는 어휘 학습 도구이며 의료 진단, 처방, 환자 안전 판단 도구가 아니다. 의학 단어 카드와 관련 화면에 학습 참고용 안내를 표시한다.
- 뜻·예문·발음은 출처와 조회 시점을 추적할 수 있게 하고, 일반 의미와 임상 문맥을 구분한다.
- AI 설명은 확정 사실처럼 표시하지 않으며 사용자 편집과 신고/수정 경로를 제공한다.
- 약어, 용량, 시술명처럼 오해 위험이 큰 항목은 자동 확장을 제한하고 신뢰 가능한 의학 사전 검증 전 “확인 필요” 상태로 둔다.
- 실제 치료나 과제 제출 전 교재, 교수자, 공인 의학 자료로 확인하도록 안내한다.

## 운영 점검

- 릴리스 전 권한 문구, 개인정보 처리방침, 데이터 삭제, RLS, 로그 redaction, API rate limit을 점검한다.
- 의존성 및 provider 약관은 정기 검토하고, 사고 대응 담당·키 회전·사용자 통지 절차를 배포 전에 문서화한다.
- 개발 fixture에는 합성 이미지와 가상 단어 목록만 사용한다.
