# 화면 상세 명세

## 공통 규칙

- 모든 화면은 한국어 UI, iPhone 세로 화면을 기준으로 하며 Android의 시스템 뒤로 가기와 edge-to-edge를 지원한다.
- 주요 CTA는 화면당 하나이며 하단 safe area 위에 둔다. 키보드가 열리면 입력 대상과 CTA가 함께 보이도록 스크롤한다.
- 터치 영역은 최소 44×44pt다. 상태를 색만으로 표현하지 않고 문구·아이콘·접근성 상태를 함께 사용한다.
- OCR 후보는 사용자가 `검토 완료`를 누르기 전 단어장에 저장하지 않는다.
- 아래 `빈 상태`가 성립하지 않는 화면은 “해당 없음” 대신 안전한 초기/데이터 없음 상태를 정의한다.

## 1. 온보딩

- **화면 목적:** 첫 사용자에게 가져오기→검토→복습 가치와 로컬 우선 원칙을 30초 안에 설명한다.
- **핵심 콘텐츠:** 3개 페이지의 짧은 제목·그림 영역·한 문장 설명, 페이지 표시, 개인정보/의료 안내 링크.
- **주요 버튼:** `다음`, 마지막 `시작하기`; 보조 `건너뛰기`.
- **사용자 행동:** 페이지 넘기기, 안내 링크 열기, 건너뛰기.
- **다음 이동 화면:** 홈. 권한 요청은 하지 않는다.
- **로딩 상태:** 로컬 정적 콘텐츠이므로 spinner 없이 즉시 표시한다.
- **빈 상태:** illustration이 없으면 아이콘과 텍스트만으로 의미가 완전해야 한다.
- **오류 상태:** 온보딩 완료 저장 실패 시 홈 진입은 허용하고 다음 실행에 다시 보지 않도록 재시도한다.
- **접근성 고려사항:** 페이지 제목부터 읽고 현재 `1/3`을 알린다. swipe 외에 버튼 탐색이 가능해야 한다.
- **iPhone 작은 화면 대응:** 장식 이미지를 축소/숨기고 제목·본문·버튼을 우선한다. 본문은 스크롤 가능하게 한다.

## 2. 홈

- **화면 목적:** 사용자가 가장 필요한 다음 행동을 한눈에 선택하게 한다.
- **핵심 콘텐츠:** 인사 대신 오늘의 복습 카드, 미완료 OCR 작업 banner, 최근 단어장 최대 3개, 로컬/오프라인 상태.
- **주요 버튼:** `단어 가져오기`; 복습이 있으면 `오늘 복습 시작`.
- **사용자 행동:** 가져오기 방식 열기, 작업 이어하기, 최근 단어장 열기, 복습 시작.
- **다음 이동 화면:** 사진 촬영, 앨범 업로드, OCR 검토, 단어장 상세, 오늘의 복습.
- **로딩 상태:** 로컬 데이터 skeleton은 짧게 사용하며 가져오기 CTA는 즉시 활성화한다.
- **빈 상태:** `첫 단어장을 만들어 볼까요?`와 `단어 가져오기`, 보조 `직접 추가`.
- **오류 상태:** 로컬 조회 실패 시 재시도와 진단 안내를 제공하고 새 작업은 데이터 안전 확인 전 막는다.
- **접근성 고려사항:** 남은 복습 수를 `오늘 복습 8개`로 읽는다. 카드 전체와 내부 버튼의 중복 focus를 피한다.
- **iPhone 작은 화면 대응:** 최근 단어장은 가로 carousel 대신 세로 3개 목록으로 표시한다.

## 3. 사진 촬영

- **화면 목적:** OCR에 적합하고 개인정보가 최소화된 페이지 사진을 촬영한다.
- **핵심 콘텐츠:** 카메라 preview, 문서 가이드, 빛 반사·기울기·개인정보 제외 안내, flash 상태.
- **주요 버튼:** 큰 `촬영`; 보조 `앨범`, `닫기`, `플래시`.
- **사용자 행동:** 구도 맞추기, flash 전환, 촬영, 앨범으로 변경.
- **다음 이동 화면:** 촬영 후 이미지 미리보기·자르기, 취소 시 이전 화면.
- **로딩 상태:** 카메라 초기화 중 검은 화면 대신 안내와 activity indicator를 표시한다.
- **빈 상태:** 카메라 장치 없음/사용 불가 시 앨범 대안을 제공한다.
- **오류 상태:** 권한 거부 화면, 카메라 시작 실패 시 `다시 열기`와 `앨범에서 선택`.
- **접근성 고려사항:** 촬영 버튼에 명확한 label, flash 상태값, 가이드 문구를 제공한다. preview 자체에 핵심 텍스트를 겹치지 않는다.
- **iPhone 작은 화면 대응:** 안내는 한 줄 banner로 축약하고 촬영·닫기 영역을 safe area 안에 고정한다.

## 4. 앨범 업로드

- **화면 목적:** 앨범의 사진/스크린샷을 사용자가 명시적으로 선택하게 한다.
- **핵심 콘텐츠:** 시스템 picker 전 사전 안내, 선택 가능한 형식과 여러 장 선택 상태(후반 MVP).
- **주요 버튼:** `사진 선택하기`; 보조 `취소`.
- **사용자 행동:** 시스템 picker 열기, 한 장 또는 후반 MVP에서 여러 장 선택, 취소.
- **다음 이동 화면:** 이미지 미리보기·자르기 또는 홈.
- **로딩 상태:** asset를 앱 저장소로 복사하는 동안 `사진 준비 중`과 장별 상태를 표시한다.
- **빈 상태:** 선택 없이 picker를 닫으면 조용히 이전 화면으로 돌아가며 오류 toast를 띄우지 않는다.
- **오류 상태:** 읽기 실패/지원하지 않는 형식은 문제 파일만 표시하고 다시 선택을 제공한다.
- **접근성 고려사항:** 시스템 제한 접근 권한 상태를 설명하고 `선택한 사진 관리` 경로를 제공한다.
- **iPhone 작은 화면 대응:** 여러 장 목록은 썸네일+파일 상태의 세로 행으로 표시하고 긴 파일명은 2줄 제한한다.

## 5. 이미지 미리보기·자르기

- **화면 목적:** 단어 영역만 남겨 OCR 품질과 개인정보 보호를 높인다.
- **핵심 콘텐츠:** 이미지, crop frame, 회전, 확대/축소, `원본은 기본적으로 기기에 저장` 안내.
- **주요 버튼:** `이 이미지 사용`; 보조 `다시 촬영/선택`, `회전`.
- **사용자 행동:** crop 조정, 확대, 회전, 이미지 교체.
- **다음 이동 화면:** OCR 처리 중; 여러 장이면 다음 이미지 확인.
- **로딩 상태:** 고해상도 decode/저장 중에는 조작을 잠시 막고 진행 상태를 표시한다.
- **빈 상태:** asset가 사라졌으면 `사진을 다시 선택해 주세요`와 재선택 버튼.
- **오류 상태:** crop 저장 실패 시 원본은 보존됐음을 알리고 재시도/원본 사용을 제공한다.
- **접근성 고려사항:** crop gesture만 강제하지 않고 `왼쪽/오른쪽 5% 줄이기`, 회전 버튼 등 대안을 고려한다.
- **iPhone 작은 화면 대응:** 편집 도구를 한 줄 3개 이하로 두고 이미지 영역 최소 높이를 보장한다.

## 6. OCR 처리 중

- **화면 목적:** 긴 처리의 현재 단계와 데이터 안전을 설명한다.
- **핵심 콘텐츠:** 썸네일, `이미지 준비 중/단어 찾는 중/결과 정리 중`, 장 번호, 로컬 보존 안내.
- **주요 버튼:** 보조 `취소`; background 처리가 안전할 때 `나중에 계속`.
- **사용자 행동:** 기다리기, 취소 확인, 앱을 나갔다가 돌아오기.
- **다음 이동 화면:** OCR 결과 검토, OCR 실패, 홈의 미완료 작업.
- **로딩 상태:** 실제 단계 기반 progress를 쓰고 근거 없는 퍼센트는 표시하지 않는다.
- **빈 상태:** 처리 대상이 없으면 홈으로 복구하고 새 가져오기를 제안한다.
- **오류 상태:** 장별 실패와 전체 실패를 구분하며 성공한 장은 유지한다.
- **접근성 고려사항:** 단계가 바뀔 때 과도하지 않게 알리고 spinner에 `단어를 찾는 중` label을 준다.
- **iPhone 작은 화면 대응:** 장식 animation보다 상태 제목·취소 버튼을 우선하고 가로 진행 목록을 쓰지 않는다.

## 7. OCR 결과 검토

- **화면 목적:** 자동 저장 전 사용자가 모든 후보를 확인·교정·선택하게 한다.
- **핵심 콘텐츠:** 원문 strip, 선택 수, 전체 선택, `WordCandidateRow` 목록, 낮은 신뢰도/중복 배지, 직접 추가.
- **주요 버튼:** `{n}개 단어 검토 완료`; 보조 `단어 추가`, `모두 선택/해제`.
- **사용자 행동:** 후보 선택, 철자 수정, 삭제/제외, 원문 위치 보기, 새 단어 추가, 여러 장 전환.
- **다음 이동 화면:** 단어장 생성/선택, 단어장 상세; 뒤로 가면 임시 저장 확인.
- **로딩 상태:** 후보를 로컬에서 불러오는 skeleton. 단어 정보 조회는 이 화면의 완료를 막지 않는다.
- **빈 상태:** `찾은 단어가 없어요`와 `이미지 다시 확인`, `직접 단어 추가`.
- **오류 상태:** 일부 후보 decode 실패는 해당 행에 표시하고 나머지 검토를 허용한다. 저장 실패 시 편집 내용을 유지한다.
- **접근성 고려사항:** checkbox와 편집 버튼 label에 단어를 포함하고, 낮은 신뢰도와 중복을 텍스트로 읽는다.
- **iPhone 작은 화면 대응:** 원문 strip은 접을 수 있고 행 편집은 별도 sheet로 열어 키보드와 CTA 충돌을 막는다.

## 8. 단어장 생성

- **화면 목적:** 검토한 단어를 저장할 새 단어장의 최소 정보를 받는다.
- **핵심 콘텐츠:** 이름 입력, 분야 선택(`일반/수능/토익/간호·의학`), 선택 단어 수, 기존 단어장 선택 링크.
- **주요 버튼:** `단어장 만들기`; 보조 `기존 단어장에 저장`.
- **사용자 행동:** 이름 입력, 분야 선택, 기존 단어장 전환.
- **다음 이동 화면:** 생성된 단어장 상세 또는 기존 단어장 선택 sheet.
- **로딩 상태:** 로컬 저장 중 버튼에 `저장 중`을 표시하고 중복 제출을 막는다.
- **빈 상태:** 이름은 최근 분야를 기반으로 제안하되 사용자가 확인해야 한다.
- **오류 상태:** 이름 누락/중복은 입력 아래 설명, DB 실패는 후보 보존 안내와 재시도.
- **접근성 고려사항:** 분야 chip은 radio group으로 읽고 선택 상태를 알린다.
- **iPhone 작은 화면 대응:** 키보드 위에 CTA를 두며 분야는 2열 grid 대신 세로 또는 wrapping chip으로 배치한다.

## 9. 단어장 목록

- **화면 목적:** 모든 로컬 단어장을 찾고 학습 또는 관리한다.
- **핵심 콘텐츠:** 이름, 분야, 단어 수, 오늘 복습 수, 최근 학습일; 검색과 정렬은 후반 MVP.
- **주요 버튼:** `새 단어 가져오기`; 각 행의 `열기` 동작.
- **사용자 행동:** 단어장 열기, 새로 만들기, 후반에 검색/정렬/삭제.
- **다음 이동 화면:** 단어장 상세, 가져오기 sheet, 단어장 생성.
- **로딩 상태:** 로컬 skeleton 3행, 탭과 CTA는 유지한다.
- **빈 상태:** `아직 단어장이 없어요`와 `사진으로 시작`, `직접 만들기`.
- **오류 상태:** 목록 조회 재시도; 데이터 손상 의심 시 새 쓰기를 막고 복구 안내.
- **접근성 고려사항:** 행 label에 이름·분야·단어·복습 수를 묶고 더보기는 별도 label을 준다.
- **iPhone 작은 화면 대응:** 카드보다 밀도 낮은 세로 list를 사용하고 메타데이터는 최대 2줄이다.

## 10. 단어장 상세

- **화면 목적:** 한 단어장의 상태를 보고 단어를 추가·열람·학습한다.
- **핵심 콘텐츠:** 이름/분야, 단어·복습 수, `학습 시작`, 검색, 단어 목록, 출처/확인 필요 상태.
- **주요 버튼:** 복습이 있으면 `오늘 {n}개 복습`, 없으면 `플래시카드 학습`; 보조 `단어 추가`.
- **사용자 행동:** 학습, 단어 열기, 수동 추가, 사진 가져오기, 후반에 필터/삭제.
- **다음 이동 화면:** 플래시카드, 오늘의 복습, 단어 상세, 가져오기 sheet.
- **로딩 상태:** header는 유지하고 단어 목록만 skeleton.
- **빈 상태:** `이 단어장은 비어 있어요`와 `단어 가져오기`, `직접 추가`.
- **오류 상태:** enrichment 실패 단어에는 행 배지, 목록 실패에는 재시도.
- **접근성 고려사항:** 발음기호를 강제로 읽히지 않게 별도 label을 제공하고 상태 배지는 텍스트화한다.
- **iPhone 작은 화면 대응:** header action을 2개 이하로 보이고 나머지는 더보기 menu에 둔다.

## 11. 플래시카드 학습

- **화면 목적:** 단어를 회상하고 답을 확인해 학습한다.
- **핵심 콘텐츠:** 진행 `3/20`, 앞면 단어/선택 이미지, 뒤면 뜻·품사·발음·예문·연상 설명·출처.
- **주요 버튼:** 앞면 `답 보기`; 일반 학습은 뒤면 `다음`, 복습 모드는 네 단계 평가.
- **사용자 행동:** 카드 뒤집기, 발음 재생, 다음, 중단/재개.
- **다음 이동 화면:** 다음 카드, 완료 요약, 오늘의 복습/단어장 상세.
- **로딩 상태:** 다음 카드 데이터를 미리 읽고 이미지는 placeholder와 출처 텍스트를 유지한다.
- **빈 상태:** 학습할 활성 단어가 없으면 단어장 상세로 안내한다.
- **오류 상태:** 이미지/발음 실패가 카드 전체를 막지 않으며 텍스트로 계속 학습한다.
- **접근성 고려사항:** 뒤집힌 상태를 알리고 gesture 외에 버튼을 제공한다. 평가 버튼은 순서와 예상 간격을 읽는다.
- **iPhone 작은 화면 대응:** 카드 내부를 스크롤 가능하게 하되 평가 바는 safe area 위에 고정한다.

## 12. 연상 이미지 선택

- **화면 목적:** 무료 검색 결과에서 출처가 분명하고 기억에 맞는 이미지를 선택한다.
- **핵심 콘텐츠:** 검색어, 2열 결과 grid, 선택 미리보기, 작가·provider·라이선스·원본 링크.
- **주요 버튼:** `이 이미지 사용`; 보조 `검색어 수정`, `이미지 없이 계속`, 향후 조건부 `AI 이미지 만들기`.
- **사용자 행동:** 검색, 후보 선택, 출처 상세 확인, 검색 결과 부적합 표시.
- **다음 이동 화면:** 단어 상세 또는 카드 미리보기; 실패 시 이미지 검색 실패 상태.
- **로딩 상태:** grid skeleton과 검색 중 label. 기존 결과는 새 검색 성공 전 유지한다.
- **빈 상태:** `알맞은 이미지를 찾지 못했어요`와 검색어 수정, 이미지 없이 계속. 향후에만 AI 생성.
- **오류 상태:** 네트워크 오류와 라이선스 누락을 구분하며 누락 이미지는 선택 불가.
- **접근성 고려사항:** 이미지 alt는 검색어+작가, 선택 상태와 출처 버튼을 읽는다. 색만으로 선택 표시하지 않는다.
- **iPhone 작은 화면 대응:** 2열을 유지하되 출처는 선택 후 하단 sheet에서 전체 표시한다.

## 13. 단어 상세

- **화면 목적:** 단어의 학습 정보를 확인하고 사용자 기준으로 수정한다.
- **핵심 콘텐츠:** 단어, 품사, 뜻, 발음/재생, 예문, 연상 설명, 이미지/출처, 학습 상태, 의학 주의.
- **주요 버튼:** `수정`; 보조 `이미지 선택`, `복습에서 제외`, `삭제`.
- **사용자 행동:** 필드 편집, 발음 재생, 이미지 변경, 출처 열기, 삭제.
- **다음 이동 화면:** 편집 sheet, 이미지 선택, 단어장 상세.
- **로딩 상태:** 로컬 필드는 즉시, 외부 enrichment 필드만 skeleton/재시도.
- **빈 상태:** 비어 있는 필드마다 `뜻 직접 입력`, `이미지 추가`처럼 개별 행동을 둔다.
- **오류 상태:** 필드별 `불러오지 못함`; 저장 실패 시 편집 draft 보존.
- **접근성 고려사항:** 섹션 heading 구조, 발음 버튼 label, 출처 link 역할, 의료 주의 우선 읽기.
- **iPhone 작은 화면 대응:** 모든 섹션을 한꺼번에 편집하지 않고 필드별 sheet를 사용한다.

## 14. 퀴즈

- **화면 목적:** 보기 중 뜻을 고르는 방식으로 회상 수준을 확인한다.
- **핵심 콘텐츠:** 진행률, 문제 단어, 4개 이하 보기, 제출 후 설명과 단어 상세 링크.
- **주요 버튼:** 선택 전 비활성 `정답 확인`, 결과 후 `다음 문제`.
- **사용자 행동:** 답 선택, 확인, 설명 보기, 중단.
- **다음 이동 화면:** 다음 문제, 퀴즈 결과, 단어 상세.
- **로딩 상태:** 로컬 문제 생성 중 짧은 상태. 빈 보기를 임의로 표시하지 않는다.
- **빈 상태:** 보기 생성에 필요한 단어가 부족하면 플래시카드를 추천한다.
- **오류 상태:** 문제 생성 실패 시 해당 단어를 건너뛰고 기록에 오답으로 남기지 않는다.
- **접근성 고려사항:** 보기는 radio group, 정답/오답은 아이콘·문구·알림으로 전달한다.
- **iPhone 작은 화면 대응:** 보기를 세로로 배치하고 긴 뜻은 줄바꿈한다.

## 15. 오늘의 복습

- **화면 목적:** 오늘 예정된 카드 수와 예상 부담을 보여 주고 복습을 시작한다.
- **핵심 콘텐츠:** 남은/완료 카드, 예상 시간, 단어장별 수, 이어서 할 세션, 완료 상태.
- **주요 버튼:** `복습 시작` 또는 `복습 이어하기`.
- **사용자 행동:** 전체 시작, 단어장별 시작, 완료 기록 보기.
- **다음 이동 화면:** 플래시카드 복습, 완료 요약, 통계.
- **로딩 상태:** 로컬 일정 계산 중 skeleton; 마지막 계산 시각을 숨길 필요는 없다.
- **빈 상태:** `오늘 복습을 마쳤어요`와 `자유 학습`, 또는 신규 사용자는 `첫 카드 만들기`.
- **오류 상태:** 일정 계산 실패 시 자유 학습은 허용하되 복습 평가는 저장 안전 확인 전 막는다.
- **접근성 고려사항:** progress를 값과 문장으로 제공하고 chart 없이도 남은 수를 이해하게 한다.
- **iPhone 작은 화면 대응:** 단어장별 카드는 세로 목록, 큰 원형 chart는 보조 요소로 축소한다.

## 16. 학습 통계

- **화면 목적:** 사용자가 학습량과 기억 변화를 부담 없이 이해하게 한다.
- **핵심 콘텐츠:** 오늘/7일 카드 수, 기억률, 연속 학습일, 어려운 단어 목록, 지표 정의.
- **주요 버튼:** `어려운 단어 복습`; 기간 선택은 segmented control.
- **사용자 행동:** 기간 전환, 어려운 단어 학습, 지표 도움말 보기.
- **다음 이동 화면:** 플래시카드, 단어 상세, 학습 탭.
- **로딩 상태:** 로컬 집계 skeleton, chart axis는 유지한다.
- **빈 상태:** `아직 학습 기록이 없어요`와 `첫 학습 시작`.
- **오류 상태:** 일부 집계 실패 시 가능한 숫자만 표시하지 말고 해당 카드에 재계산 상태를 둔다.
- **접근성 고려사항:** chart마다 문장 요약과 데이터 table 대안을 제공한다.
- **iPhone 작은 화면 대응:** 한 화면에 chart 하나씩 세로 배치하고 가로 스크롤을 피한다.

## 17. 설정

- **화면 목적:** 사진, 데이터, 학습, 화면, 안내 정책을 사용자가 통제하게 한다.
- **핵심 콘텐츠:** 원본 자동 삭제/보존, 테마, 모션, 온보딩 다시 보기, 권한 상태, 데이터 삭제, 의료 안내, 앱 정보.
- **주요 버튼:** 화면 전체 CTA 없음; 위험 영역의 `모든 로컬 데이터 삭제`.
- **사용자 행동:** toggle/선택 변경, 시스템 설정 열기, 안내 보기, 데이터 삭제.
- **다음 이동 화면:** 시스템 설정, 온보딩, 상세 안내/확인 dialog.
- **로딩 상태:** 로컬 설정 즉시 표시. 저장 중에는 해당 행 상태만 갱신한다.
- **빈 상태:** 지원하지 않는 설정은 숨기며 빈 section을 보이지 않는다.
- **오류 상태:** 저장 실패 시 이전 값으로 되돌리고 이유와 재시도를 알린다.
- **접근성 고려사항:** toggle label과 현재 값, 위험 동작의 확인 문구, 동적 테마 변경 알림.
- **iPhone 작은 화면 대응:** 설명이 긴 행은 제목 아래 여러 줄, trailing control은 줄어들지 않게 한다.

## 18. 권한 거부

- **화면 목적:** 거부를 비난하지 않고 현재 가능한 대안과 복구 방법을 준다.
- **핵심 콘텐츠:** 필요한 권한, 필요한 이유, 현재 상태, 설정 변경 단계.
- **주요 버튼:** 영구 거부면 `설정 열기`; 보조 `앨범에서 선택` 또는 `홈으로`.
- **사용자 행동:** 설정 이동, 대안 선택, 돌아가기.
- **다음 이동 화면:** 시스템 설정, 앨범 업로드, 홈.
- **로딩 상태:** 앱이 foreground로 돌아올 때 권한 재확인만 표시한다.
- **빈 상태:** 권한 종류를 알 수 없으면 일반 설정 안내보다 안전한 홈 이동을 제공한다.
- **오류 상태:** 설정 열기 실패 시 수동 경로를 텍스트로 안내한다.
- **접근성 고려사항:** 장식용 금지 아이콘을 숨기고 제목부터 focus한다.
- **iPhone 작은 화면 대응:** 설정 단계는 3줄 이하, CTA는 세로로 쌓는다.

## 19. OCR 실패

- **화면 목적:** 실패 종류를 설명하고 원본을 잃지 않은 채 다시 진행하게 한다.
- **핵심 콘텐츠:** 썸네일, `찾지 못함/처리 실패/일부 실패`, 보존 상태, 개선 팁 최대 3개.
- **주요 버튼:** `이미지 다시 확인` 또는 `다시 시도`; 보조 `직접 단어 추가`, `나중에 계속`.
- **사용자 행동:** crop 재조정, 재시도, 수동 추가, 작업 보관.
- **다음 이동 화면:** 미리보기·자르기, OCR 처리, 단어장 상세/수동 추가, 홈.
- **로딩 상태:** 재시도 후 OCR 처리 화면으로 명확히 전환한다.
- **빈 상태:** 원본도 없으면 재촬영/앨범 선택만 제공한다.
- **오류 상태:** 반복 실패 시 같은 메시지만 반복하지 않고 직접 추가 대안을 강조한다.
- **접근성 고려사항:** 원인과 행동을 읽고 팁 이미지에 의존하지 않는다.
- **iPhone 작은 화면 대응:** 팁을 접을 수 있게 하고 대안 버튼을 화면 하단에 유지한다.

## 20. 네트워크 오류

- **화면 목적:** 로컬 기능과 온라인 기능의 차이를 알려 작업 중단을 최소화한다.
- **핵심 콘텐츠:** 연결 상태, 실패한 온라인 기능(뜻/이미지 등), 로컬에 보존된 내용.
- **주요 버튼:** `다시 연결`; 보조 `나중에 계속`, `직접 입력`.
- **사용자 행동:** 재시도, 오프라인으로 저장/학습, 설정 확인.
- **다음 이동 화면:** 원래 화면으로 복귀하거나 홈/단어 상세.
- **로딩 상태:** 재연결 확인 중 버튼 상태만 갱신하고 편집 내용을 유지한다.
- **빈 상태:** 연결이 복구되면 성공 toast 후 원래 task를 한 번만 재개한다.
- **오류 상태:** 반복 429/서버 오류는 네트워크 없음과 구분해 `잠시 후`를 안내한다.
- **접근성 고려사항:** banner가 나타날 때 한 번 알리고 focus를 강제로 빼앗지 않는다.
- **iPhone 작은 화면 대응:** 전체 화면보다 상단 banner/해당 섹션 inline 오류를 우선한다.

## 21. 빈 상태 패턴

- **화면 목적:** 데이터가 없는 이유와 첫 행동을 명확히 해 막힌 느낌을 줄인다.
- **핵심 콘텐츠:** 간단한 아이콘, 구체적 제목, 한 문장 가치 설명, CTA 하나, 필요할 때 보조 행동.
- **주요 버튼:** 맥락별 `단어 가져오기`, `직접 단어 추가`, `첫 학습 시작`.
- **사용자 행동:** 첫 데이터 생성, 대안 선택, 도움말 확인.
- **다음 이동 화면:** 가져오기 sheet, 수동 추가, 플래시카드.
- **로딩 상태:** 데이터 조회가 끝나기 전에 빈 상태를 잠깐 보여 주지 않는다.
- **빈 상태:** 자체 패턴이며 단어장 없음/검색 결과 없음/복습 완료/통계 없음 문구를 구분한다.
- **오류 상태:** 조회 실패를 빈 상태로 위장하지 않는다. 오류 문구와 재시도를 사용한다.
- **접근성 고려사항:** 장식 아이콘은 숨기고 제목→설명→CTA 순서로 읽는다.
- **iPhone 작은 화면 대응:** 세로 중앙 고정 대신 상단에서 80~120pt 아래에 배치해 큰 글자 스크롤을 허용한다.
