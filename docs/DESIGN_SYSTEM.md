# 디자인 시스템

## 경험 원칙

LinkVoca의 화면은 “촬영 기술”보다 “틀린 단어를 안심하고 고치는 경험”을 앞세운다. 한 화면에 한 가지 주요 행동만 강조하고, OCR·사전·이미지 결과는 자동 정답이 아니라 수정 가능한 제안으로 표현한다. iPhone의 한 손 조작과 Dynamic Type을 우선하되 Android에서도 동일한 의미와 상태를 유지한다.

- 기능 밀도보다 제목→현재 상태→다음 행동이 분명한 정보 위계를 우선한다.
- 보조 정보와 고급 조작은 필요한 상태에서만 펼치는 Progressive Disclosure를 적용한다.
- 한 화면에서 강한 Primary Action은 하나만 사용한다. 나머지 행동은 링크나 중립 표면으로 낮춘다.
- 상태 배지는 오류·확인 필요처럼 즉시 행동이 필요한 정보부터 최대한 적게 노출한다.
- 복잡한 보조 기능은 기본 화면에 나열하지 않고 펼침 영역, 메뉴 또는 해당 상태의 인라인 행동으로 제공한다.

## 시각 방향

- 브랜드 인상: 연결, 발견, 차분한 학습. 과도한 게임화와 의료 앱처럼 보이는 차가운 표현을 피한다.
- 기본 배경 `#F7F8FA`, 표면 `#FFFFFF`, 주요 텍스트 `#17202A`
- Primary `#3563E9`: 주요 CTA, 선택, 학습 진행
- Secondary `#18A58B`: 확정, 완료, 정답에 가까운 피드백
- Warning `#D97706`: OCR 검토 필요, 출처 확인
- Danger `#D64545`: 삭제, 처리 실패
- 색상은 제안 토큰이며 구현 전 WCAG 대비를 검증한다. 상태를 색만으로 전달하지 않고 아이콘과 문구를 함께 쓴다.

## 타이포그래피와 간격

- iOS 시스템 글꼴을 기본으로 하고 한국어와 영어 혼합 시 줄 높이를 충분히 둔다.
- 화면 제목 28/34 semibold, 섹션 제목 20/26 semibold, 본문 16/24 regular, 보조 14/20 regular, 카드 단어 32/38 bold를 출발점으로 한다.
- Dynamic Type 확대에서 텍스트를 고정 높이로 자르지 않는다. 발음기호와 품사는 줄바꿈을 허용한다.
- 간격은 4pt 배수(`4, 8, 12, 16, 24, 32`)를 사용하고 화면 좌우 기본 여백은 20pt로 한다.
- 터치 영역은 최소 44×44pt를 확보한다.

## 핵심 컴포넌트

- **PrimaryButton:** 화면당 원칙적으로 하나. “OCR 시작”, “단어 확정”, “복습 시작”처럼 결과가 분명한 동사를 쓴다.
- **WordCandidateRow:** 원문 후보, 신뢰도 표식, 편집, 제외 토글을 제공한다. 낮은 신뢰도는 자동 선택 해제하지 않고 검토 배지를 표시한다.
- **SourceImageStrip:** OCR 원문 위치와 후보를 연결해 오타를 비교할 수 있게 한다.
- **WordInfoCard:** 단어, 품사, 뜻, 발음, 예문, 연상 설명을 섹션별로 수정 가능하게 한다.
- **ImageResultCard:** 썸네일과 함께 제공자·작가·출처 링크를 표시하고 라이선스 정보가 없으면 선택 불가로 처리한다.
- **ReviewCard:** 앞면은 단어/이미지 힌트 설정에 따르고, 뒤집기 전 뜻이 노출되지 않게 한다.
- **RatingBar:** `다시`, `어려움`, `보통`, `쉬움` 순서와 예상 다음 간격을 보여 준다.
- **StatusBanner:** 오프라인, 부분 실패, 동기화 대기 상태를 작업을 막지 않는 방식으로 설명한다.

## 화면별 원칙

- 촬영 화면은 페이지 테두리와 빛 반사/기울기 안내를 제공하되 자동 촬영은 MVP에서 제외한다.
- OCR 교정 화면은 일괄 선택과 개별 편집을 모두 제공하고, 뒤로 가기 시 임시 작업을 보존할지 묻는다.
- 단어 정보 화면은 외부 조회 실패 필드만 재시도할 수 있어야 한다.
- 이미지 선택 화면은 무료 검색 결과를 먼저 보여 주고, AI 생성은 빈 상태 또는 “적합한 이미지 없음” 이후의 보조 행동으로 둔다.
- 복습 화면은 평가 직후 다음 카드로 이동하되 실행 취소를 짧게 제공한다.

## 접근성 및 국제화

- VoiceOver/TalkBack label에는 아이콘 모양이 아니라 행동과 대상을 쓴다.
- 카드 뒤집기, OCR 완료, 오류는 시각 효과뿐 아니라 접근성 알림으로 전달한다.
- 모션 감소 설정을 존중하고 뒤집기 애니메이션 없이도 앞/뒤 상태를 구분한다.
- 한국어 설명 UI를 기본으로 하되 영어 단어, IPA, 예문의 선택/복사가 가능해야 한다.
- 의학 용어는 일반 의미와 의학 의미를 라벨로 구분하고 “학습 참고용” 안내를 가까이 표시한다.
